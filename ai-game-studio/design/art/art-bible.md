**版本**: 0.1

## 一句话美术定义

> Spellchef 的视觉语言是**「会动的魔法食谱绘本」**——温暖的米白底色承托高饱和的食材色彩，圆润拟人的角色从页面里跳出来，每一个技能特效都像在厨房里发生的小小奇迹。

---

## 1. 视觉风格

### 核心关键词

温暖 / 扁平立体 / 食材拟人 / 魔法感 / 绘本质感

### 风格锚点

介于 *Hades* 的精致度与 *Cuphead* 的表现力之间，底色是绘本插画质感。

| 参照 | 借鉴维度 | 不借鉴维度 |
|------|----------|------------|
| Hades | 俯视角比例、动作反馈手感 | 写实暗黑风格 |
| Cuphead | 夸张表情、帧动画表现力 | 橡皮管卡通 + 噪点滤镜 |
| Ghibli 食物场景 | 高饱和暖色、食物质感 | 写实细节描绘 |
| Overcooked | 食材可爱化处理方向 | 低多边形低精度 |

### 造型语言规则

- 所有角色和食材：**圆角优先**，消除所有尖锐直角（除非刻意表达危险/邪恶）
- 描边：统一使用暖棕色 `#4A2E1A`，线宽随距离镜头远近等比缩放
- 材质：2D 手绘线条 + 轻微水彩晕染纹理，非像素风非 3D 建模

## 3. 角色设计规范

### 主角（法厨）

**比例**: 头身比 1:2.5，大头圆眼，手脚略短胖

**标志性元素**:
- 厨师帽（帽顶有当前主属性的微型符文发光，随属性切换变色）
- 锅铲作为主武器，平时插在腰带上
- 围裙作为血量视觉反馈：
  - 100-75% HP：围裙完好
  - 75-50% HP：出现污渍
  - 50-25% HP：边缘破损
  - 25-0% HP：焦黑破烂

**动态要求**:
- 移动：有"弹弹的"惯性（出发和刹车时额外帧）
- 技能释放：蓄力→爆发的夸张帧，体现烹饪动作（挥锅铲、甩食材、翻炒手势）
- 翻滚：身体压扁→伸展的弹性形变，留下短暂残影

### 怪物设计规范

**核心原则**: 所有怪物都是「被诅咒的食材/厨具」拟人化，非传统奇幻怪兽。

**怪物类型层级**:

| 类型 | 设计语言 | 体型比例 | 轮廓形状（俯视角） |
|------|----------|----------|------------------|
| 普通小怪 | 单一食材拟人，表情委屈/愤怒，圆滚滚 | 主角×0.5 | 圆形 |
| 精英怪 | 两种食材融合，有厨具配件（刀片翅膀、锅盖护盾） | 主角×1.5 | 方形/六边形 |
| Boss | 完整"料理"拟人，邪恶厨师造型，充满房间 | 房间×0.4 | 不规则多边形 |

**可读性规则**: 怪物轮廓形状必须在俯视角 3 格外可辨认（小怪圆、精英方、Boss 不规则）

**怪物示例参考**:
- 普通：愤怒的茄子、暴走的蒜头、沸腾的汤团
- 精英：辣椒+章鱼 → 烤章鱼辣爆兵（配件：辣椒刀翅）
- Boss：第8层最终Boss — 黑暗主厨（全身炭焦食材拼装，手持巨型焦黑锅铲）

### 食材视觉设计

**拟人规则**:
- 所有食材必须有眼睛和微表情
- 掉落状态：闭眼休眠
- 被拾取：惊喜睁眼
- 被合成：开心表情（进入锅的瞬间）

**属性传达（双重编码，色盲友好）**:
- 颜色光晕（见属性色表）
- 图案纹理：火=火焰纹 / 冰=雪花纹 / 雷=锯齿纹 / 毒=螺旋纹 / 治愈=十字纹 / 土=叶脉纹

## 4. UI 视觉规范

### 整体风格

**手绘羊皮纸 + 现代信息层级**：仿旧绘本装饰边框 + 清晰的现代字体，装饰服务于可读性，不凌驾于信息之上。

### HUD 布局

```
┌─────────────────────────────────────────┐
│  [HP心形] [背包预览2格]                 │  ← 左上：生命 + 食材
│                                         │
│           [ 游戏主视野 ]                │
│                                         │
│  [技能槽①][②][③][④]    [套餐效应]   │  ← 底部：操作 + 状态
└─────────────────────────────────────────┘
```

**HP 条**: 绘本风心形分段（5颗心），满血=红心，受伤=灰心，不使用数字进度条

**技能槽**:
- 圆形卡牌样式，圆角矩形变体
- 菜名在悬停/键盘聚焦时弹出气泡提示
- 冷却用模拟时钟遮罩（顺时针清除，非线性进度条）
- 活跃状态：金色边框发光

**套餐效应指示**:
- 右下角小盾牌图标
- 激活时：金色高亮 + 效果名单行文字 + 粒子装饰
- 未激活时：灰色图标，不抢视觉焦点

### 菜单/界面风格

**配方书/背包界面**:
- 模拟真实食谱书：绑线装订、翻页动效、纸张质感
- 未发现配方：墨水污渍遮挡（非"???"文字）
- 食材拖拽进锅：锅有咕嘟咕嘟预热动画
- 配方首次解锁：金色光芒从墨水污渍中扩散消散

**通用弹窗规则**:
- 圆角 `12px`，暖棕描边 `3px`
- 背景：米白 `#F5F0E8` + 轻微纸张纹理
- 阴影：暖棕半透明 `rgba(74,46,26,0.2)`

### 字体规范

| 用途 | 字体风格 | 特征 |
|------|----------|------|
| 标题/菜名 | 手写圆体（参考 Nunito Bold） | 活泼清晰，支持中文 |
| 数值/数据 | 等宽无衬线 | 数字对齐，快速阅读 |
| 故事/配方描述 | 手写衬线（模拟钢笔字） | 氛围感，用于非关键信息 |

**可读性要求**: 所有 HUD 文字最小字号 14px，在 1080p 分辨率下 1 米距离可辨认

## 7. 视觉层级原则

**俯视角场景中的注意力引导优先级**:

1. **玩家角色**（始终最亮、最高对比度）
2. **可拾取食材**（金色/彩色光晕，轻微弹跳动画）
3. **危险攻击区域**（红色预警标记，3帧前出现）
4. **怪物**（中等对比度，不压过食材）
5. **环境/地面**（最低对比度，做"画布"而非"主角"）

**视觉噪音控制规则**:
- 同屏特效不超过 15 个独立粒子系统
- 非关键怪物不使用闪光/发光效果
- 关卡环境图案：低饱和度，与角色色彩系保持 40%+ 的明度差

---

## 8. 禁止清单（Anti-Patterns）

- ❌ 任何现实主义血腥/内脏表现（食物被击败用爆星星/彩虹，非血）
- ❌ 高对比度纯黑背景（底色始终米白系）
- ❌ 超过3种颜色共存的单一角色（色彩混乱）
- ❌ 在非 Boss 场景使用满屏特效
- ❌ 中文文字使用像素字体（可读性差）
- ❌ 怪物外形与食材来源无关联
