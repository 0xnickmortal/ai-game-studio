# Spellchef — 核心玩法机制 GDD

**版本**: 0.1
**日期**: 2026-03-28
**负责人**: game-designer
**状态**: 已批准

---

## 1. 概述（Overview）

Spellchef 的核心玩法是「边战斗边烹饪」的三层 Build 构筑系统，运行在俯视角 Roguelite 框架下。玩家在战斗中收集食材（Layer 1），在关卡内的烹饪站合成魔法菜肴（Layer 2），通过装备槽中的菜组合触发套餐效应（Layer 3）。三层决策嵌套形成完整的战略深度：短期拾取决策、中期配方决策、长期 Build 方向决策。

---

## 2. 玩家幻想（Player Fantasy）

玩家应感到自己是一名**即兴发挥的天才法厨**——不是照食谱做饭的厨师，而是在战场废墟中捡起食材、当场悟出配方的魔法创作者。每次 Build 成型的瞬间（尤其是套餐效应首次触发时）应产生「啊，原来这样能连起来！」的发现感。

**目标 MDA Aesthetics**：Discovery（首位）> Expression > Challenge > Fantasy

---

## 3. 核心循环（三层嵌套）

### 微循环（约 30 秒）
```
移动/闪避 → 攻击怪物 → 怪物掉落食材 → 拾取决策（要/不要）→ 继续战斗
```
每一次拾取都是轻量决策：这个食材和我现有的能凑成什么配方？背包满了要不要丢弃旧食材？

### 中循环（5-15 分钟，一个楼层）
```
清空普通房间 × 2-3 → 烹饪站：合成新菜 → 装备槽调整 → 套餐效应评估 → 精英房间 → 宝藏房间 → 进入下一层
```
每层结束时玩家的 Build 方向应更清晰，通过「烹饪站」节点给予战略喘息时间。

### 宏循环（单局 30-60 分钟）
```
选起始厨师 → 选起始道具 → 逐层 Build 成型（第1-3层基础期，第4-6层成型期，第7层精英挑战）→ 第8层最终头目 → 获得厨师之星 → 局外解锁
```

## 4. 操作方式

| 输入 | 行为 | 备注 |
|------|------|------|
| WASD | 移动（八方向） | |
| 空格 | 翻滚（无敌帧 0.3s） | 冷却 1.5s |
| 左键 | 释放装备槽1的菜 | |
| 右键 | 释放装备槽2的菜 | |
| Q | 释放装备槽3的菜 | |
| R | 释放装备槽4的菜 | |
| E | 拾取食材 / 与烹饪站交互 | 拾取范围 0.8 单位 |
| Tab | 打开背包与套餐面板 | 暂停时间（PvE 单人） |
| 1234 | 快速聚焦装备槽（高亮，不切换） | 可选的高级操作 |

**设计意图**：四个装备槽全部可以主动释放（非自动触发），保留玩家操作感。翻滚是核心闪避工具，不设无限翻滚，逼迫玩家学习怪物攻击节奏。

## 5. 规则系统

### 5.1 食材系统（Layer 1）

**属性矩阵**

| 元素 | 符号 | 主效果方向 | 克制关系 |
|------|------|------------|----------|
| 火 | 🔥 | 持续灼烧伤害 | 克冰 |
| 冰 | ❄ | 减速/冻结 | 克土 |
| 雷 | ⚡ | 范围/连锁 | 克水（未来扩展属性） |
| 毒 | ☠ | 叠层减益 | 克治愈系怪物 |
| 治愈 | 💚 | 回血/护盾 | 无克制，自成体系 |
| 土 | 🌿 | 控场/减速AOE | 克毒 |

**品质等级**

| 品质 | 掉落层级 | 数值倍率 |
|------|----------|----------|
| 普通 | 第1-6层 | ×1.0 |
| 稀有 | 第4-7层 | ×1.6 |
| 传说 | 第7层专属 | ×2.5 |

**背包规则**
- 默认上限：6个食材
- 天赋扩展上限：8个（需解锁）
- 超出上限时拾取新食材会触发「替换」提示，玩家选择丢弃哪一个

### 5.2 配方系统（Layer 2）

**合成规则**

| 食材数量 | 结果 | 效果描述 |
|----------|------|----------|
| 2个食材 | 基础菜 | 单一效果，数值标准 |
| 3个食材 | 强化菜 | 单一效果强化版，或主效果+副效果 |

**烹饪站交互流程**
1. 玩家走近烹饪站，按 E 交互
2. 打开烹饪界面（时间暂停）
3. 从背包拖入 2-3 个食材到锅中
4. 预览结果菜名与效果
5. 确认 → 2秒烹饪动画 → 生成一道菜
6. 菜直接进入装备槽（若槽满，提示选择替换或丢弃）

**配方书机制**
- 初始状态：所有配方词条显示为「???」
- 首次合成该配方 → 词条解锁，显示完整效果与菜名
- 配方书跨局保存（局外进度的一部分）
- 设计意图：鼓励实验，第一次发现配方的「啊哈时刻」是核心乐趣之一

**配方组合逻辑示例**（供实现参考，完整表见 `design/gdd/recipe-book.md`）

| 食材组合 | 菜名 | 效果 |
|----------|------|------|
| 🔥 + 🔥 | 炎爆浓汤 | 发射3颗火球，每颗造成灼烧 |
| 🔥 + ❄ | 蒸汽刺身 | 直线蒸汽喷射，击退+灼烧 |
| 💚 + 💚 + 💚 | 圣光全席 | 大范围治愈+3秒护盾 |
| ☠ + 🌿 + 🌿 | 毒雾麻婆 | 在地面留下持续毒雾区域 |

### 5.3 套餐效应系统（Layer 3）

套餐效应由玩家**装备槽中当前装备的菜**组合决定，实时计算，换菜即时更新。

**触发规则**

| 条件 | 效果名 | 效果 |
|------|--------|------|
| 同属性 2 道菜 | 专精入门 | 该属性伤害+25%，状态效果时长+50% |
| 同属性 3 道菜 | 专精大师 | 在「专精入门」基础上：该属性技能有15%概率冷却立即重置 |
| 4槽全不同属性 | 融合厨师 | 每道菜释放时，额外附带一次随机属性爆发（伤害为基础值50%） |
| 特定菜组合 | 隐藏效果 | 游戏内不提示，自行发现或查攻略 |

**隐藏套餐效应设计原则**
- 数量：首批约8-12个隐藏套餐
- 触发条件：2-3道特定命名菜同时装备
- 效果强度：比普通套餐强约30-50%，属于奖励性而非必需性
- 传播设计：玩家发现后倾向分享，构成自然的社区传播点

## 6. 公式（Formulas）

### 6.1 基础伤害公式
```
技能伤害 = 基础伤害值 × 品质倍率 × 套餐加成倍率 × 怪物防御系数

变量定义：
  基础伤害值    由配方类型决定，存于 assets/data/recipes.json
  品质倍率      普通=1.0 / 稀有=1.6 / 传说=2.5
  套餐加成倍率  无效应=1.0 / 专精入门=1.25 / 专精大师=1.25（冷却重置单独计算）
  怪物防御系数  0.7-1.0（不同怪物类型，存于 assets/data/enemies.json）
```

### 6.2 难度曲线公式
```
第N层怪物生命值 = 基础HP × (1.05)^(N-1)
第N层怪物攻击力 = 基础ATK × (1.04)^(N-1)

每层基线上移5%（HP）和4%（ATK），保持玩家的成长感与压迫感同步增长
```

### 6.3 TTK 校验公式
```
TTK（秒）= 怪物HP / (玩家DPS × 套餐加成)

目标 TTK（普通怪，无套餐加成）：8-12秒
目标 TTK（精英怪，有套餐加成）：25-40秒
目标 TTK（最终头目，满 Build）：90-120秒
```

## 7. 进度机制

### 7.1 局内进度（垂直，单局内）

```
层级  | 食材品质       | 特殊规则
------|----------------|----------------------------------
1-3层 | 普通           | 基础期，熟悉配方，建立Build方向
4-6层 | 普通+稀有(30%) | 成型期，套餐效应开始发挥作用
7层   | 仅传说         | 精英挑战层，高压，高回报
8层   | —              | 最终头目，所有套餐效应全激活状态
```

**房间结构（每层）**
```
普通战斗房 × 2-3 → 精英房（必选，包含精英怪+优质掉落）→ 宝藏房（含烹饪站）→ Boss房 / 层级出口
```

**锯齿难度节奏**
```
普通房（缓）→ 精英房（峰值）→ 宝藏房（释放）→ 下一层（基线+5%）
```

### 7.2 局外进度（横向，跨局持久）

**货币**：厨师之星（★）
- 获取：完成单局 +3★ / 首次通关 +5★ / 完成挑战条件（无伤某层、用特定 Build 通关等）+1-3★

**解锁树**

```
★ 5  → 起始厨师 B（冰系专精，被动：冰食材额外掉落率+20%）
★ 8  → 起始道具 B（便当盒：初始背包+2格）
★12  → 新菜系页：「甜点系」（新属性组合解锁）
★18  → 起始厨师 C（毒系专精，被动：毒叠层上限+2）
★25  → 挑战模式：无烹饪站（所有菜需击杀精英怪触发自动合成）
```

## 8. 边界情况（Edge Cases）

| 情况 | 处理方式 |
|------|----------|
| 背包满时拾取食材 | 弹出「替换」选择界面，暂停时间，玩家选择丢弃或取消拾取 |
| 合成结果占满所有装备槽 | 新菜进入「待装备」临时槽，玩家需手动分配或丢弃 |
| 同一层无烹饪站生成 | 保证机制：每2层至少出现1个烹饪站（程序保底） |
| 套餐效应实时冲突 | 以最后一次换菜后的状态为准，无延迟生效 |
| 0个食材进入Boss层 | 允许，但Boss层入口会有警告提示「你手中无菜，确定冒险？」 |
| 传说食材在1-3层意外掉落 | 设计上不存在（程序层禁止），但若因BUG出现，按稀有处理 |
| 翻滚冷却中受到攻击 | 正常受伤，无额外保护，鼓励玩家精确管理翻滚时机 |

**退化策略预防**：
- 「全治愈 Build」退化（4槽全装治愈菜无限回血）：治愈菜对 Boss 伤害降低60%，且 Boss 有「诅咒」状态压制治愈效果（治愈量减半）
- 「传说食材囤积」退化（不合成只囤食材）：食材上限8个严格执行，超出必须丢弃，且未合成食材无战斗效果

## 10. 调节旋钮（Tuning Knobs）

| 旋钮 | 类型 | 当前值 | 范围 | 修改理由 |
|------|------|--------|------|----------|
| 翻滚无敌帧时长 | Feel | 0.3s | 0.2-0.5s | 低于0.2s玩家感知不到；高于0.5s太安全 |
| 翻滚冷却 | Feel | 1.5s | 1.0-2.5s | 核心节奏控制，影响动作感 |
| 背包上限（默认） | Gate | 6格 | 4-8格 | 压迫感 vs 自由度平衡点 |
| 烹饪动画时长 | Feel | 2.0s | 1.0-3.0s | 给予战略感，不能太快（无感）或太慢（烦） |
| 同属性套餐加成 | Curve | 25% | 15-40% | 需测试是否过强/过弱 |
| 套餐冷却重置概率 | Curve | 15% | 10-25% | 15%约每7次触发1次，有惊喜感但不破坏平衡 |
| 层级HP成长系数 | Curve | 1.05/层 | 1.03-1.08 | 8层后总HP约1.48倍，在Build成长对冲范围内 |
| 稀有食材出现率（4-6层） | Curve | 30% | 20-45% | 确保玩家有足够稀有食材但不过于泛滥 |

## 11. 验收标准（Acceptance Criteria）

### 功能性标准
- [ ] 玩家能完成完整的食材拾取→合成→装备→套餐效应触发流程
- [ ] 配方书首次合成后正确解锁词条，跨局保存
- [ ] 套餐效应在换菜后0.1秒内实时更新（无延迟）
- [ ] 背包满时强制弹出替换界面，阻止静默丢失
- [ ] 每2层保底出现1个烹饪站

### 体验性标准（Playtest 验证）
- [ ] 第一次游玩的玩家在20分钟内能独立发现并合成至少3种不同配方
- [ ] 80%测试玩家在第一局内自然发现套餐效应（无需教程提示）
- [ ] 测试玩家平均在第4-5层时感到「Build 方向已确立」
- [ ] 隐藏套餐效应首次触发时，测试玩家有可观察到的积极反应（出声、截图等）
- [ ] 单局平均时长落在30-60分钟区间内（抽样5局取平均）
- [ ] Boss 战 TTK 落在90-120秒区间（满 Build 状态，普通难度）
